windows如何下通过Hexo搭建github静态Blog

windows如何下通过Hexo搭建github静态Blog

自己花了两个晚上熬夜尝试研究，才算是成功有了自己静态Blog的雏形。网上看了好多介绍，但是发现都很难一次性的找到所有答案，我结合了几篇文章的介绍，算是完成。现在把握总结的精简步骤在这里介绍一下，希望能够对有缘的朋友有些帮助。

一、准备工作

  1. 安装Node.js，按照默认设置即可。这个下载，网上的资源都能找到。
  2. 安装git。这个在腾讯的软件管家里就可以直接下载安装，也是默认设置。
  3. 在Windows下运行CMD，输入命令‘git –version’， 测试是否安装成功。
  4. 注册github账号，注意账户名字最好能够考虑一下以后Blog的可传播性，并结合自己的其他社交账户的名字，这样以便于日后可以多平台传播。我的微博和微信名都叫“未来之LU”，我的github账户就叫FutureLu。新建repository，名字就是username.github.io.
  5. 生成新的SSH Key。输入命令ssh-keygen -t rsa -C “XXXX@xx.com”。此处是你的邮箱地址。输入命令后，正常直接三次回车即可。
  6. 在Github里将新生产的SSH Key从id_rsa.pub粘贴到github里。
  7. 注意你要先在制定位置建立新文件夹Blog，然后选择文件夹右键—git bash.
  8. 通过git bash，CD到Blog目录，安装hexo，命令’npm install -g hexo’。安装完成后，通过命令’Hexo’测试。
  9. 初始化Hexo，命令’$ Hexo init hexo’。成功之后，在Blog文件夹里会多出一个Hexo的文件夹。
  10. 从此步开始的所有命令都是在git Bash中进行，并且要CD到Blog下的Hexo所在目录。我的安装是在E：\blog\hexo。
  11. 安装依赖文件：’npm install’。
  12. 部署形成文件：’hexo generate’或者’hexo g’。将MD文件生成静态网页
  13. 运行server：’hexo server’或者’hexo s’。//启动本地服务，进行文章预览调试。会提示http://localhost:4000，可以在浏览器中打开试试看效果。这里Hexo已经基本安装完毕。

二、配置github page

  1. 如上所述，新建的repository名字要是username.github.io

  2. 点击github头像边上的小箭头，选择setting—launch automatic page generator—continue to layouts。随后，随意选择一个模板—publish page，发布生成了一个静态站点。但是，此时的站点还没有和hexo相连通。

  3. 将本地hexo项目托管到github。打开hexo目录下的“_config,yml”，编辑后面的deploy属性，

    type: git

    repositoty: git@github.com:username/username.github.io.git

    branch: master

    还有其他的配置都可以在这里更改，以后再自己慢慢琢磨，线上线再说。

  4. git bash配置身份信息

    git config –global user.name “yourname”

    git config –global user.email “youremail”

  5. 安装hexo-deployer-git插件:‘npm install hexo-deployer-git –save’。

  6. 命令’hexo d’，正常会显示deploy done.

    OK，至此，基本上基于Hexo的github静态Blog已经搭建完毕。你需要自己新建Blog.md的文件来测试一下。

三、新建博客

  1. 输入命令：’hexo new “my new post”‘，或者任意你新建MD文件的名字；
  2. 在source文件夹中找到md文件，进行编辑，保存。
  3. hexo g
  4. hexo s //正常以后，此步骤可以不用。
  5. 如果用http://localhost:4000浏览效果后，需要control+C将server cancel.
  6. hexo d
  7. 打开username.github.io即可看见刚刚编辑的新Blog更新。

四、注意每次配置更改的流程

  1. hexo clean //清空缓存
  2. hexo generate //将MD文件生成静态网页
  3. hexo deploy //部署
  4. 然后打开你的Blog网站地址username.github.io来看看效果。

下面要学习的是安装主题，给站点添加统计和搜索功能等模块，以及配置文件的更改。。。
